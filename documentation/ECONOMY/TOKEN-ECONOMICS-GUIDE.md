# Token Economics Guide

## Executive Summary

Relay's token economy creates a revolutionary economic system that rewards authentic human connection, democratic participation, and community building rather than financial speculation or manipulation. Through carefully designed incentive structures, the system ensures that economic rewards flow to those who contribute genuine value to the network while preserving privacy and preventing extractive behaviors.

This comprehensive token ecosystem supports democratic governance, sustainable community growth, and authentic human verification through four interconnected token types: RELAY tokens for governance and transactions, Vote tokens for democratic participation, Reputation Credits for community building, and Channel tokens for local economies.

# *What this means for users:* Earn real economic value by being a good community member, participating in governance, and helping others - not through speculation or manipulation.

## Economic Philosophy

### Core Principles

**Value Creation over Speculation**
- Rewards for actual network contribution rather than financial manipulation
- Long-term incentives favor sustained positive behavior
- Anti-speculation mechanisms prevent purely extractive participation

**Privacy-Preserving Economics**
- Economic activity doesn't compromise user privacy
- Zero-knowledge proofs for reputation-based rewards
- Anonymous participation in economic activities when desired

**Community-Centric Design**
- Economic benefits flow to community builders and maintainers
- Governance tokens distributed based on positive community participation
- Collective ownership of network value and decision-making

**Sustainable Growth**
- Economic model designed for long-term network health
- Inflation mechanisms tied to actual utility and adoption
- Anti-concentration measures prevent wealth accumulation in few hands

### Vote Tokens (Two-Phase Democratic Participation)

**System Design Philosophy**
- **Phase 1 (New Users)**: Limited tokens earned through network growth and engagement
- **Phase 2 (Trusted Users)**: Unlimited voting with only channel-specific decay rules
- **Security**: Separate from invite tokens which use generational decay for Sybil resistance

**Phase 1: New User Token Earning (First ~30 Days)**
```yaml
Initial Allocation: 20 vote tokens

Earning Opportunities (No Caps - Motivation Rewarded):
  Friend Added: +5 tokens
  Channel Joined: +3 tokens  
  Daily Engagement: +2 tokens (once per day)
  Biometric Verification: +5 tokens
  Guardian Setup: +5 tokens
  Successful Invite: +2-10 tokens (random reward)

Graduation Requirements:
  - Trust Score: >= 75
  - Channels Joined: >= 3
  - Days in System: >= 7
  - Biometric Verified: True
  - Guardian Setup: True
```

**Phase 2: Trusted User (Unlimited Voting)**
```yaml
Vote Capacity: Unlimited
Earning: No longer needed
Decay: Only channel-specific vote decay parameters
Security: Flagging system for suspicious activity only

Flagged User Restrictions:
  Vote Limit: 10 tokens until trust restored
  Invite Tokens: Frozen
  Restoration: Automatic when anomaly patterns clear
```

**Invite Tokens (Separate Security System)**
```yaml
Purpose: Onboard new users (Sybil attack prevention)
Allocation: Generational decay tree (parent - 1, minimum 1)
Regeneration: No - finite resource by design
Security: One-time use, prevents mass account creation
```

**Generational Decay Mechanics**
The number of invite tokens you receive depends on your inviter's generational position:
- **Generation 0 (Founders)**: Start with 10 invite tokens
- **Generation 1**: Receive 9 invite tokens (founder - 1)
- **Generation 2**: Receive 8 invite tokens (gen 1 - 1)
- **Generation N**: Receive max(1, 10 - N) invite tokens
- **Minimum Floor**: Everyone gets at least 1 invite token

## Anti-Manipulation Mechanisms

### Preventing Economic Attacks

**Sybil Attack Prevention**
```yaml
Identity Requirements:
  - Proof of unique human identity through Guardian Recovery system
  - Social graph analysis for detecting fake accounts
  - Reputation building requirements that take significant time investment

Economic Barriers:
  - Token bonding requirements for high-privilege actions
  - Progressive penalties for detected manipulation attempts
  - Community-based validation of new account legitimacy
```

**Market Manipulation Prevention**
```yaml
Trading Restrictions:
  - Gradual token release for earned rewards (vesting schedules)
  - Anti-whale mechanisms preventing concentration
  - Transparent reporting of large token movements

Price Stability:
  - Token supply adjustments based on network usage
  - Treasury management to dampen extreme price movements
  - Focus on utility value rather than speculative trading
```

**Governance Attack Prevention**
```yaml
Voting Security:
  - Time-locked voting power to prevent flash loans attacks
  - Reputation requirements for governance participation
  - Multi-layer approval for significant protocol changes

Proposal Quality:
  - Community review requirements before formal voting
  - Economic stakes for proposal submission
  - Penalty mechanisms for spam or low-quality proposals
```

## Privacy-Preserving Economics

### Anonymous Economic Participation
- **Zero-Knowledge Rewards**: Earn tokens and reputation without revealing personal information
- **Anonymous Transactions**: Economic activity can be completely private while maintaining fraud prevention
- **Reputation Privacy**: Build reputation without exposing the specific activities that earned it
- **Selective Disclosure**: Choose when and how to reveal economic achievements

### Economic Privacy Protections
- **No Financial Surveillance**: Economic activity patterns cannot be used to infer personal information
- **Confidential Balances**: Token holdings are private unless voluntarily disclosed
- **Anonymous Governance**: Vote and participate in governance without revealing identity
- **Private Contributions**: Support causes and communities without exposure of political or social preferences

### Anti-Exploitation Economics
- **No Manipulation Markets**: Token systems designed to prevent pump-and-dump schemes and manipulation
- **Authentic Value Creation**: Economic rewards tied to genuine community contribution rather than financial engineering
- **Anti-Speculation Mechanisms**: Long-term incentives favor sustained positive behavior over short-term extraction
- **Community Protection**: Economic design prevents wealthy actors from capturing disproportionate influence

### Wealth Distribution Safeguards
- **Anti-Concentration Measures**: Diminishing returns prevent excessive wealth accumulation
- **Democratic Access**: Everyone can earn tokens through contribution regardless of initial financial resources
- **Progressive Mechanics**: System designed to reduce rather than amplify existing economic inequalities
- **Community Ownership**: Network value accrues to participants rather than external shareholders

# *Economic benefit:* Earn real value through positive community contribution while maintaining complete economic privacy and protection from exploitation.
