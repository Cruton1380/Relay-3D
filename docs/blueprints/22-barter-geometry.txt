══════════════════════════════════════════════════════════════
  TRANSFERPACKET AS SPECIALIZATION
══════════════════════════════════════════════════════════════

  TransferPacket ⊂ ExchangePacket

  TransferPacket:
    All legs share one unit type
    balanceCondition = net_zero_per_unit
    Traditional double-entry bookkeeping

  ExchangePacket (categorical):
    Legs can use different unit types
    balanceCondition = all_legs_fulfilled
    Multi-dimensional barter

  SAFETY CONSTRAINTS
══════════════════════════════════════════════════════════════

  1. Explicit legs (no implicit obligations)
  2. Explicit template-declared units
  3. Explicit settlement window
  4. Explicit consolidation rule
