## **THE HEAVY/LIGHT PATTERN (SYSTEMS DESIGN)**

### **Not Theology**

This is constraint-satisfaction structure, NOT universal truth.

### **The Pattern**

**Heavy (constraint, inertia, persistence)**:
- **Down dimension** ‚Üí History, replay, canon
- Committed state
- High cost to change
- Stable, reliable

**Light (exploration, hypothesis, possibility)**:
- **Outward dimension** ‚Üí Speculation, drafts, proposals
- Uncommitted state
- Low cost to change
- Fluid, uncertain

**Balanced (coordination)**:
- **Surface dimension** ‚Üí Active work, negotiation
- Transitional state
- Medium cost to change
- Dynamic equilibrium

### **Engineering Precedent**

**Same pattern appears in**:
- Simulated annealing (high temp = exploration, low = commitment)
- Git workflow (branch = light, main = heavy)
- Machine learning (high learning rate = exploration, low = convergence)
- Optimization algorithms (exploration vs exploitation)

**This is systems design, not theology.**

### **Core Features (Week 3-4)**

1. **Context Breadcrumbs**
   - Show path history (where you've been)
   - Click to navigate back to any context
   - Visual trail of your work

   - After interruption (15+ min away)
   - Shows last context, task, recent activity
   - "Jump Back" restores exact state

   - Visual path on 3D globe
   - Fading trail (recent = bright, old = faint)

   - Not just browser back (URL history)

   - Restores files, cursor, task, scroll

   - Captures context before you leave
   - Triggers on tab switch, inactivity, close

   - See divergence points

8. **Mental State Indicators**
